## Overview

The Limits SDK provides a type-safe interface to the Limits Policy Evaluation API. You can:

- **Evaluate policies** with structured conditions (e.g. amount, currency) or with interaction text (e.g. LLM responses).
- **List, approve, and decline** human approval requests for escalated policies.

Authentication uses an API key (Bearer token). The SDK uses native `fetch` (Node.js 18+).

## Requirements

| Requirement | Version |
|-------------|---------|
| Node.js     | `>= 18.0.0` (for native `fetch`) |
| TypeScript  | `>= 5.0` (optional, for types)   |

## Installation

Install the Limits SDK using your preferred package manager:

<CodeGroup>
```bash npm
npm install @limits/sdk
```

```bash yarn
yarn add @limits/sdk
```

```bash pnpm
pnpm add @limits/sdk
```
</CodeGroup>

## Configuration

### Constructor: `new Limits(config)`

Create a single client instance and reuse it for all calls.

| Parameter        | Type      | Required | Description |
|------------------|-----------|----------|-------------|
| `config.apiKey`  | `string`  | **Yes**  | API key. Must start with `sk_`. |
| `config.debug`   | `boolean` | No       | Enable `[LimitsSDK]` console logs. Default: `false`. |

**Throws:** `InvalidAPIKeyError` if the key is missing, not a string, doesn't start with `sk_`, or is too short.

```typescript
import { Limits } from '@limits/sdk';

const limits = new Limits({
  apiKey: process.env.LIMITS_API_KEY!,
  debug: process.env.NODE_ENV === 'development',
});
```

### Authentication

- Header: `Authorization: Bearer <apiKey>`
- API keys must start with `sk_` and be valid in your Limits account.

## Get Your API Key

1. Sign up at [app.limits.dev](https://app.limits.dev)
2. Click on the **User Menu** (your profile section with the person icon at the bottom left of the dashboard)
3. In the settings modal that opens, click on **API Keys** (with the `<>` icon) in the left sidebar
4. Click **+ Generate API Key** to create a new API key
5. Add it to your environment variables

```bash
export LIMITS_API_KEY="sk_your_api_key_here"
```

<Warning>
  Never commit your API key to version control. Use environment variables or a secrets manager.
</Warning>

## Quick Example

```typescript
import { Limits } from '@limits/sdk';

const limits = new Limits({
  apiKey: process.env.LIMITS_API_KEY!,
  debug: true, // Optional: enable [LimitsSDK] logging
});

// Check policy with structured data (conditions)
const result = await limits.check('amount-minimum-usd', {
  amount: 500,
  currency: 'USD',
});

if (result.isBlocked) {
  console.error('Blocked:', result.data.reason);
} else if (result.isEscalated) {
  console.warn('Escalated:', result.data.reason);
} else {
  console.log('Allowed:', result.data.reason);
}
```
