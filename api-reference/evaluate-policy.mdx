---
title: "Evaluate policy"
description: "Overview of policy evaluation endpoints (conditions, instructions, guardrails)"
---

The API provides three evaluation endpoints, one per policy mode:

| Endpoint | Mode | Request body | Use case |
|----------|------|--------------|----------|
| [POST /api/policies/{policyKey}/evaluate/conditions](/api-reference/evaluate-conditions) | Conditions | `request.input` (object) | Structured rules (amount, user, resource). |
| [POST /api/policies/{policyKey}/evaluate/instructions](/api-reference/evaluate-instructions) | Instructions | `request.prompt`, `request.input` (string) | Prompt + LLM response validation. |
| [POST /api/policies/{policyKey}/evaluate/guardrails](/api-reference/evaluate-guardrails) | Guardrails | `request.input` (string) | Text safety, PII, moderation. |

All three return the same shape: `action` (ALLOW, BLOCK, or ESCALATE), `success`, `message`, and optional `details` or `error`.
