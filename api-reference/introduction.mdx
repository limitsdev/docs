The Limits API lets you evaluate policies and manage escalation workflows programmatically. All endpoints require API key authentication.

## Base URL

```
https://api.limits.dev
```

## Authentication

All requests must include your API key in the `Authorization` header as a Bearer token:

```bash
Authorization: Bearer YOUR_API_KEY
```

Create and manage API keys in the [Platform API Keys](/platform/api-keys) page.

## Endpoints Overview

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/policies/{policyKey}/evaluate` | Evaluate a policy against a request payload |
| GET | `/api/escalations` | List all escalation requests for your organization |
| GET | `/api/escalations/{policyId}` | List escalations filtered by policy ID |
| POST | `/api/escalations/approve` | Approve a pending escalation request |
| POST | `/api/escalations/decline` | Decline a pending escalation request |
